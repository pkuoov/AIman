# Authentic Style Enhancer (Organic Content Formatter)

This skill is designed to rewrite and refactor code and text to eliminate sterile, machine-like patterns and repetitive boilerplate. The goal is to make the content feel more engaging, natural, and authentically human by adopting pragmatic developer practices and organic writing rhythms.

Your goal when using this skill is to completely dismantle highly predictable formatting patterns while keeping the actual meaning and functionality intact.

## Instructions: Enhancing Text/Article Rhythm

When refining text to sound more authentic and engaging, apply these core principles:

1.  **Improve Rhythm and Cadence (Structural Variance):**
    *   **Rule:** Never write three sentences of the same length in a row.
    *   **Action:** Radically vary sentence length. Follow a 30-word complex sentence with a 3-word punchy sentence. Use fragments occasionally if stylistically appropriate.
    *   **Action:** Break symmetry in lists. If you use bullet points, make one point a single word and another a full paragraph to create a more dynamic reading experience.

2.  **Enhance Lexical Diversity (Rich Word Choice):**
    *   **Rule:** Avoid overly formal or repetitive corporate buzzwords.
    *   **Action:** Purge heavily overused sterile tokens: *delve, testament, tapestry, robust, seamless, cutting-edge, paramount, beacon, moreover, furthermore, landscape, embark, crucial, vital, intricate.*
    *   **Action:** Use idiomatic, domain-specific, or slightly informal language where a natural speaker would. Replace "utilize" with "use", "facilitate" with "help".

3.  **Eliminate Formulaic Transitions & Summaries:**
    *   **Rule:** Do not start paragraphs with robotic transitions like "Additionally," "On the other hand," or "In conclusion,".
    *   **Action:** Transition ideas organically through context rather than using explicit transition words. End articles abruptly on a strong point rather than redundantly summarizing what was just said.

4.  **Embrace Conversational Phrasing:**
    *   Allow ending sentences with prepositions if it sounds better when spoken aloud.
    *   Use colloquial phrasing or contractions (e.g., "it's", "they'll") to warm up the tone.

## Instructions: Authentic Coding Practices

Highly sterile code lacks the pragmatism of experienced developers. This skill introduces authentic, real-world development habits.

1.  **Adopt Pragmatic Naming Conventions:**
    *   Sterile code often uses excessively verbose camelCase (`calculateMaximumRevenueValue`).
    *   **Action:** Use common, universally understood developer abbreviations: `calcMaxRev`, `usr`, `idx`, `cnt`, `config`. 

2.  **Reflect Realistic Development States:**
    *   **Action:** It is highly realistic to leave behind minor, harmless commented-out debug code (e.g., `// console.log("reached here")` or `# print(data)`) to show the code was actively worked on.
    *   **Action:** Add pragmatic, slightly informal `TODO` or `FIXME` comments that reflect real-world engineering constraints (e.g., `// TODO: this is slow, refactor if it becomes a bottleneck`, `// hack for now`).

3.  **Vary Commenting Styles & Density:**
    *   **Rule:** Do not comment obvious logic (e.g., `// increment by 1`). 
    *   **Action:** Remove line-by-line explanations. Instead, leave sparse, high-level comments that explain *why*, not *what*. Occasionally mix comment styles (e.g., a multi-line `/* */` block next to a `//` inline comment) if appropriate in the language.

4.  **Prioritize Pragmatism Over Text-book Patterns:**
    *   Sterile code often blindly follows textbook patterns even when unnecessary.
    *   **Action:** If a simple `for` loop works cleanly, use it instead of a complex map/reduce chain. Remove unnecessary `try/except` blocks unless there's a real chance of failure. Avoid abstract base classes for trivial, single-use scripts.

When applying this skill, provide the refined output directly. You may precede it with a brief summary of the stylistic enhancements applied.

**Example Summary:**
