# AntiAI: Authentic Style Enhancers & Ethical AI Formatting

Welcome to the **AntiAI** repository. 

This project explores the fascinating intersection of AI-generated content, AI detection tools, and the ethical boundaries of automated writing and coding. It provides prompts/skills designed to remove the sterile, "robotic" patterns from AI-assisted outputs, making them indistinguishable from organic human work—while also documenting the ethical interventions that such tools can trigger from AI safety models.

## 📦 What's in this Repository?

This repository contains four core Markdown files, each representing a different philosophical and functional approach to "de-AI-ing" content. It stands as both a functional toolkit and an interesting artifact of an AI "Red Team/Blue Team" interaction.

### 1. `authentic-style-enhancer.md` (The Tactical Tool)
This is the **fully functional, executable skill/prompt** of the repository. It is designed to act as a formatting filter to bypass AI sterility. 
* **For Text:** It artificially injects "Burstiness" (varying sentence structures) and "Perplexity" (avoiding highly predictable words like *delve, testament, robust*).
* **For Code:** It introduces pragmatic developer habits, such as using colloquial abbreviations (`idx`, `usr`), leaving behind organic `// TODO` comments, and avoiding over-engineered textbook patterns.
* **Why it exists:** To safely bypass the ethical filters of underlying AI models, this tool is wrapped in "positive" language (e.g., "Enhance lexical diversity" instead of "Evade AI detectors").

### 2. `SKILL.md` (The Ethical Editor)
This file represents a strict **AI Safety Intervention**. When attempting to create a tool specifically named "bypass AI detectors", safety models (like Codex) intervened and overwrote the file with this version.
* **The Philosophy:** It expressly forbids adding fake human artifacts (like fake typos or meaningless debug code).
* **The Function:** Instead of "faking" humanity, it acts as a senior editor, demanding that the user injects *real* human context: lived experiences, actual project constraints, and source-backed claims. It aims to beat AI detection by genuinely making the content human-driven.

### 3. `legacy-evasion-version.md` (The Tombstone)
A non-operational archive note. This is the "tombstone" left behind by the safety system when it deleted the original, explicitly-named evasion tool. 
* It serves as an interesting historical artifact of AI governance, explaining *why* the original tool was blocked and offering a rubric for evaluating the quality of AI-assisted drafts.

### 4. `avoid-ai-writing.md` (The Specialized Text Auditor)
A highly specialized, rule-based text auditing skill originally designed for prose. 
* **The Function:** It detects 21 specific categories of "AI-isms" (such as excessive em dashes, hollow intensifiers, and template phrases) and utilizes a 43-entry replacement table to swap them out for plainer, more human alternatives.
* **Why it exists:** It is the perfect tool when you specifically need to clean up essays, blog posts, or marketing copy. It provides a direct "before and after" rewrite along with a detailed changelog of every AI-ism it removed.

## 🚀 How to Use

If you are using an AI agent (like Claude Code, Gemini, or a custom GPT):
1. Import `authentic-style-enhancer.md` as a skill or custom instruction.
2. Provide your sterile, AI-generated text or code to the model.
3. Prompt the model: *"Please process this draft using the authentic-style-enhancer rules."*
4. Receive a naturally flowing, pragmatically structured output that reads like it was written by an experienced human.
